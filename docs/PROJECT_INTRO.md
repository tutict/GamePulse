# GamePulse 项目介绍

GamePulse 是一个本地优先的游戏社区评论分析工具，面向游戏运营、制作和社区团队，用来把多平台玩家反馈转成可审查、可追溯、可行动的舆情报告。

它的第一版目标不是做一个大而全的爬虫平台，而是先打通从“收集当前可见评论”到“生成证据化分析报告”的完整闭环。

## 解决什么问题

游戏版本更新、角色上线、活动调整或 BUG 爆发后，玩家反馈通常分散在 B站、Steam、NGA、Reddit、TapTap、小黑盒等社区里。人工阅读可以发现个别问题，但很难稳定回答这些问题：

- 玩家整体情绪有没有变差？
- 高频抱怨集中在哪些主题？
- 某个版本是好评更多，还是 BUG 和劝退更多？
- 哪些角色或系统正在被频繁讨论？
- BUG 反馈能不能按问题类型聚类？
- 玩家是否出现退款、退坑、弃坑等流失风险信号？

GamePulse 把这些问题拆成结构化数据、聚类、证据样例和中文报告，让团队能更快判断哪些反馈需要处理、哪些需要继续验证。

## 核心能力

- 多来源接入：支持 CSV/JSON 导入、Steam/Reddit 连接器，以及面向 B站、NGA、TapTap、小黑盒等页面的当前可见内容采集脚本。
- 本地优先：评论数据、报告和 API Key 默认留在本机或本地 Docker 栈中。
- 证据回溯：报告里的结论会关联原文摘录、平台和来源链接。
- 玩家情绪分析：识别正向、中立、负向和混合情绪。
- 高频抱怨聚类：按性能、崩溃、数值、付费、内容、剧情、角色等主题聚合。
- BUG 聚类：识别闪退、黑屏、卡死、报错等问题信号。
- 版本评价：按项目配置的版本发布时间，比较版本窗口内的评论表现。
- 角色热度：通过“词表 + AI 候选”的方式跟踪角色、系统、玩法模式等实体。
- 流失风险指数：基于退坑、退款、弃坑、劝退等文本信号生成舆情风险指标，不伪装成真实留存预测。

## 当前产品形态

GamePulse 由几个部分组成：

- Web Dashboard：用于创建项目、导入数据、启动分析、查看报告和搜索证据。
- Local API：提供项目、导入、评论搜索、分析任务和报告接口。
- Worker：执行分块分类、聚类和报告生成。
- Postgres + pgvector：保存项目、评论、标签、聚类和报告，为后续向量检索预留空间。
- Redis + BullMQ：承载本地分析任务队列。
- Userscript / Playwright 采集脚本：把当前页面可见评论提交到本地 API。

本地访问地址：

- Web Dashboard: `http://127.0.0.1:5173`
- API Health: `http://127.0.0.1:4317/api/health`

## 典型工作流

1. 在看板里创建一个游戏项目，例如《绝区零》。
2. 配置版本窗口、Reddit 关键词、Steam AppID、角色和系统别名。
3. 通过 CSV/JSON、连接器或当前页面采集脚本导入评论。
4. 启动分析任务。
5. 查看中文报告，重点关注风险指数、高频抱怨、BUG 聚类、角色热度和原文证据。
6. 对高风险簇回到原平台验证上下文，再决定公告、修复、客服话术或后续数据验证。

## 架构原则

- 先验证分析闭环，再扩展采集自动化。
- 不保存 cookies，不做后台大规模隐式爬取。
- 评论作者默认脱敏，保留原文、时间、平台和链接用于审查。
- 分析任务使用分块处理和结果缓存，避免百万级历史库下重复计算。
- 大模型只作为可配置能力接入，默认启发式管线可以零成本运行。

## 已验证样例

当前实现已经用《绝区零》做过一次端到端测试：

- 创建项目和版本窗口。
- 导入覆盖 B站、TapTap、NGA、Reddit、小黑盒等来源形态的测试评论。
- 从 TapTap 公开评价页采集当前可见评价。
- 生成包含风险指数、抱怨聚类、BUG 聚类、角色/系统热度和证据链接的中文报告。

这次测试也暴露了下一步要改进的点：长评通常包含多个主题，纯启发式分类会把部分证据归得偏粗，后续需要引入更稳定的模型化多标签分类和簇摘要。

## 当前限制

- Reddit 无 OAuth 的公开 JSON 请求可能被返回 403，后续应支持 OAuth 或手动导入。
- 国内平台的稳定采集优先通过当前页面脚本完成，不保证后台自动拉取。
- 启发式分类适合 MVP 验证，不等同于最终生产质量。
- 版本评价依赖项目配置的发布时间和评论时间；历史数据时间缺失时只能做弱分析。
- 流失风险是舆情风险，不是基于真实活跃、留存、付费数据的预测模型。

## 下一步

- 增加模型化多标签分类，降低长评误聚类。
- 优化当前页面采集器的 UI，把项目 ID、采集数量和提交结果展示得更清楚。
- 支持 Reddit OAuth 和更多官方/低风险连接器。
- 增加报告导出、对比报告和人工修正标签。
- 引入嵌入向量检索，用于相似评论、相似 BUG 和历史版本对比。
